# 古典导聆目录站点框架方案

## Summary
- 站点定位为“公益性的古典音乐版本导聆目录”，核心不是流媒体播放器，而是可持续更新的作品索引与版本推荐库。
- 公开站点采用“静态优先”架构：内容来自本地结构化数据，构建后输出静态页面，便于长期低成本维护与迁移。
- 视觉与信息架构参考 [Piano Library 首页](https://www.pianolibrary.org/)、[作曲家索引](https://www.pianolibrary.org/composers/)、[About](https://www.pianolibrary.org/about/)：整体保持简洁、文字优先、留白充足、弱装饰、强层级。
- 已有离线资料可作为首批数据源：当前大致为 8 位作曲家、15 个作品目录、约 205 个版本页；现有元数据以资源链接、演出信息、图片为主，专辑/发行商/发行日期应设计为可选字段。

## Key Changes
- 技术框架定为 `Astro + TypeScript` 公开站点，数据层用 `YAML/JSON + Zod 校验`，搜索使用预构建索引加前端分层检索；部署目标保持静态托管兼容，不绑定特定平台。
- 维护架构定为“双层结构”：一个公开站点 + 一个仅本机运行的小型本地 Web 维护工具。维护工具负责增改曲目、人物、版本、链接、图片与排序，不直接依赖线上后台。
- 内容模型固定为 6 类实体：
  - `Composer`：中外文名、年代、简介、别名、排序键。
  - `Person`：指挥、独奏家、歌手、组合、乐团等统一建模；角色可多值。
  - `WorkGroup`：作品分组树，支持任意深度路径，如“交响曲”或“奏鸣曲 / 小提琴奏鸣曲”。
  - `Work`：作品标题、原文标题、作品号、简介、所属作曲家、所属分组路径。
  - `Recording`：推荐版本主体，包含排序、主推荐标记、封面/演出照、演出与专辑信息、资源链接、备注。
  - `ResourceLink` / `Credit`：外链与演出参与者的子结构。
- 生成索引固定为 5 份：
  - `stats`：作曲家数、作品数、版本数、指挥人数、最近更新时间。
  - `composerTree`：作曲家页所需的“类型优先”树。
  - `conductorIndex`：按指挥聚合推荐版本。
  - `personIndex`：供搜索与版本页跳转使用的人物索引。
  - `searchIndex`：只收录作曲家、类型、作品、指挥/演奏者；默认不直接把具体版本作为首层结果。
- 页面结构固定如下：
  - 首页：站点标题、每日推荐式随机版本区块（按日期种子稳定轮换）、收录统计、最近更新、快速入口。
  - 作曲家总览页：作曲家列表与快速筛选。
  - 作曲家详情页：先显示类型树，再逐层进入作品；不直接铺满版本。
  - 指挥浏览页：指挥列表；进入后展示其相关推荐版本，按作曲家/作品分组。
  - 搜索页或全局搜索栏：分组结果显示“作曲家 / 类型 / 作品 / 指挥与演奏者”。
  - 作品页：作品简介、基础资料、推荐版本列表。
  - 版本页：图片、完整演出信息、时间地点、专辑与发行信息、外部资源链接。
  - About 页：项目说明、公益声明、更新方式、联系方式、勘误与建议入口。
- 版本页字段要求固定为：
  - 图片：封面、艺术照、现场照可多图，允许为空。
  - 演出信息：乐团、指挥、歌手、独奏家、组合均支持多个条目。
  - 时间地点：拆分为日期文本、地点文本，允许不完整。
  - 专辑信息：专辑名、发行商、发行日期均为可选。
  - 资源链接：支持 bilibili、YouTube 与其他站点，多链接并存。
- 本地维护工具固定提供 4 类能力：
  - 新增与编辑作曲家、人物、作品、版本。
  - 自动生成 slug、排序键、页面路径与索引更新。
  - 对必填字段、重复项、失效关系、空链接做校验。
  - 预览单条记录会生成哪些页面，并触发站点重建。
- 一次性导入策略固定纳入首期实现：
  - 从现有 `rar/chm` 提取目录树与 HTML 内容。
  - 自动迁移作曲家简介、人物简介、作品简介、版本页链接与图片路径。
  - 对无法可靠解析的字段生成待人工补录清单，而不是静默丢失。

## Test Plan
- 数据层：非法角色、缺失作曲家引用、重复版本路径、空资源链接、坏图片路径都应在构建前报错。
- 索引层：同一作品在“作曲家浏览”“指挥浏览”“搜索结果”中路径一致，且始终先经过类型层。
- 首页：同一天推荐结果稳定，日期变化后自动轮换；统计数字与内容库一致。
- 搜索：输入“贝多芬”“交响曲”“卡拉扬”“安妮·费舍尔”时，首层返回的分别是正确的分组实体，而不是杂乱版本列表。
- 作品页：推荐版本按设定排序显示，点击进入版本页后信息完整可读。
- 版本页：多参与者、多图片、多外链、缺失专辑信息、缺失地点信息都能正常渲染。
- 导入脚本：用现有资料抽样验证作曲家页、作品页、版本页的迁移结果，并输出人工复核报告。

## Assumptions
- UI 语言以中文为主，保留原文名与英文名字段用于检索和详情展示。
- 顶部导航首批只显式提供“首页 / 作曲家 / 指挥 / About / 搜索”；独奏家与歌手先作为搜索与版本页入口保留，不单独做顶级浏览入口。
- 首页除随机推荐外，增加“最近更新”区块，借鉴 Piano Library 的更新可见性。
- 公开站点不存储音视频文件本体，只展示说明和跳转链接。
- 线上不提供登录后台；所有写入操作只在本机维护工具中完成。
