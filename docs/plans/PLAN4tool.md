# 外链修复、图片自动化与维护工具重构方案

## Summary
- 在原方案基础上新增“图片自动选取与本地保存”能力，并把搜索目录页统一成固定高度的标准化条目布局。
- 交付顺序仍分两阶段：
  - 第一阶段：修复版本外链导入与显示、补上外链审计、统一搜索条目尺寸、补好图片字段与本地资源管线。
  - 第二阶段：重构维护工具，加入自动检查、候选审查、日志、回滚、合并建议与图片抓取选择。
- 自动更新仍采用“混合检索后端 + 规则优先 + 人工复核 + 人工确认后应用”的安全模式，不允许网络抓取结果直接覆盖正式数据。

## Key Changes
- 版本外链修复：
  - 修复 CHM 导入器对超链接的解析，强制保存 `href` 为 `url`、保存锚文本为 `title`，并对 `youtube/bilibili` 做平台一致性校验。
  - 版本页外链区改为平台按钮式展示，只显示 `bilibili`、`YouTube`、`Apple Music` 等站点名，不显示 URL 本体，不显示标题正文；按钮为可点击超链接，新窗口打开。
  - 若同一平台有多个外链，按按钮并列展示；若平台未知，显示 `其他资源` 按钮，并把原始标题保留为内部元数据供维护工具查看。

- 图片自动选取与本地保存：
  - 为人物、乐团/组合、版本分别建立图片候选与落地流程。
  - 人物与团体：
    - 优先从 `Wikipedia/Wikidata/Wikimedia Commons` 获取主图候选，因为元数据较稳定、来源可追踪。
    - 自动保存到本地资源目录，不直接热链外站；同时记录来源 URL、抓取时间、原始标题、许可/来源说明。
    - 选择规则固定为：正方或近正方优先、清晰头像优先、无遮挡优先、单人或完整团体照优先、分辨率最低阈值达标。
  - 版本：
    - 优先抓专辑封面、CD 封面、官方发行图；来源以流媒体元数据、官方唱片页、可结构化页面为主。
    - 若存在多个候选，优先选择“正面封面、构图完整、分辨率更高、与专辑标题更匹配”的图片。
  - 所有自动抓取图片都只先进入候选变更，不直接写正式数据；候选中要显示缩略图、来源、尺寸、评分与替换前后对比。
  - 数据层增加媒体元数据，而不只保留一个裸 `avatarSrc`：
    - `imageSrc`
    - `imageSourceUrl`
    - `imageSourceKind`
    - `imageAttribution`
    - `imageUpdatedAt`
    - 版本图片延续现有 `images[]`，但也要记录来源元数据。
  - 本地文件命名固定为稳定 slug + hash，避免重名覆盖；删除或回滚时只移除未被其他条目引用的图片文件。

- 维护工具重构：
  - 工具升级为 4 个主视图：
    1. `搜索与详情`
    2. `自动检查`
    3. `候选审查`
    4. `日志与回滚`
  - 自动检查主界面增加类别勾选：
    - 作曲家
    - 指挥
    - 乐团
    - 艺术家
    - 版本
  - 版本检查必须支持范围限定：
    - 按作曲家
    - 按作品
    - 按指挥
    - 按艺术家
  - 图片检查作为自动检查的一部分，但在 UI 中单独显示“信息候选”和“图片候选”两个结果分区，允许分别确认。
  - 候选审查页需要支持：
    - 字段级 diff
    - 图片前后对比
    - 合并建议
    - 单条应用
    - 批量应用
    - 忽略
    - 单条回滚
    - 整次 run 回滚
  - 自动检查 run 必须生成完整日志：
    - 扫描范围
    - 使用了哪些来源
    - 命中了哪些候选
    - 改了哪些字段
    - 哪些图片被替换
    - 哪些条目不确定
    - 哪些条目建议合并
    - 为什么不确定

- 自动检索与匹配策略：
  - 人物/团体信息：
    - 规范名、别名、拉丁转写、原文名、简称共同参与检索与打分。
    - 仅在当前简介为空时才生成简介候选，不覆盖已有简介。
    - 同人疑似项只生成 `merge proposal`，不自动合并。
  - 版本外链：
    - 平台适配器负责返回候选链接与候选封面。
    - 匹配优先级固化为：`作品号` > `作曲家+作品名` > `指挥` = `乐团` > `时间/地点` > `发行商`。
    - 只把高分且元数据不冲突的结果标记为“可直接采用”，其他全部进入人工审查。
  - 图片选择评分规则固定化，避免实现者临场拍板：
    - 分辨率得分
    - 宽高比得分
    - 人物/团体主体清晰度得分
    - 来源可信度得分
    - 与实体标题或专辑标题的命中得分
    - 水印/裁切/拼贴惩罚
    - 重复或近似重复图片惩罚

- 搜索目录页布局统一：
  - 当前 `DirectoryBrowser` 行高随摘要长度变化，导致同页条目高度不一致；新版统一为固定高度条目。
  - 每条目录项固定为三列结构：
    - 左侧：大尺寸图片区，图片高度接近整条目高度。
    - 中间：名称与基础事实区。
    - 右侧：摘要与代表作品区。
  - 所有条目使用统一高度；过长摘要和代表作品做截断，不允许撑高整行。
  - 图片尺寸调整为约等于条目高度，采用 `object-fit: cover`，无图时用同尺寸占位图。
  - 移动端不保持桌面三列，但仍保持同一列表中的卡片高度规则一致，不允许出现个别条目明显更高。
  - 搜索页与作曲家/指挥目录页共用同一条目渲染规范，避免两套样式分叉。

## Test Plan
- 外链与导入：
  - 增加 CHM 导入回归测试，覆盖“锚文本是视频标题、href 才是真链接”的情况。
  - 增加外链审计测试：`platform=url` 不一致、`url` 非 HTTP、标题被误当 URL、平台按钮渲染错误都必须报错或进入待审。
  - 增加版本页渲染测试：页面只显示平台按钮，可点击，不显示 URL 本体。
- 图片自动化：
  - 人物与团体图片选择测试覆盖“多候选时按评分选中最佳图”“低分图不自动应用”“抓取失败不污染正式数据”。
  - 版本封面测试覆盖“同版本多候选封面”“无封面时保留旧图或空图”“图片来源元数据被正确记录”。
  - 回滚测试覆盖“替换图片后单条回滚”和“整次 run 回滚后恢复旧图片引用”。
- 维护工具：
  - 自动检查测试覆盖 run 生成、候选落盘、日志生成、单条应用、批量应用、单条回滚、整次回滚。
  - 合并建议测试覆盖“别名保留、旧引用不丢失、未确认前不改正式数据”。
- 搜索目录页：
  - 组件测试或截图测试验证同页条目高度一致、图片区尺寸统一、长摘要不会拉高单条目。
  - 桌面与移动端都要做布局回归，确保图片放大后不破坏现有分栏与快速索引栏。

## Assumptions
- 图片一律本地保存，不直接热链外站。
- 人物/团体图片默认优先选结构化来源，版本封面默认优先选官方发行图或结构化流媒体元数据图。
- 自动抓到的图片和外链都先进入候选审查，不直接覆盖正式数据。
- 搜索目录页允许摘要和代表作品被严格截断，以换取条目高度完全统一。
- 若图片许可信息无法稳定拿到，工具至少要保留来源 URL 与来源站点，并在日志中明确标记该图片的许可状态未知。
